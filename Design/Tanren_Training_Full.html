Fichier unifié : Tab Entraînement (hub), Workout Builder (create + edit), Plan Builder manuel (create + edit), AI Plan Generator (prompt + generating + preview + refine).

Scope de ce fichier

20 écrans uniques × 2 thèmes = 40 mockups. Fichier autonome, aucune référence externe.

Basé sur les audits WORKOUT_BUILDER_CURRENT.md et PLAN_BUILDER_CURRENT.md. Les pain points identifiés dans les audits sont adressés dans le design (draft persistence, edit mode, drag-and-drop, validation, day values 1=Monday, etc.).

Changement clé (feedback utilisateur) : tous les inputs numériques (Sets, Reps, Poids, Repos) passent en tap-only. Aucun +/− nulle part. Chaque cellule ouvre le keyboard natif approprié au tap. Gain : plus d'espace, cohérence visuelle totale, valeurs plus lisibles (18px mono).

Training Tab · Hub — Light

Decisions for Training tab

Bloc "Aujourd'hui" en tête. Ce qui compte le plus, mis en évidence : c'est le workout à faire aujourd'hui. Border accent, corner brand, meta en uppercase (4 exos · 60 min · dernière il y a 3 j) qui donne du contexte utile avant de démarrer. Le CTA "Démarrer la séance" est primary accent, impossible à manquer.

Schedule 7 jours en grille compacte. Pas une liste verticale comme actuellement — une grille de 7 cells (Lun à Dim) avec :

→ Filled accent = jour avec workout assigné

→ Outline + "Repos" = jour libre

→ Border 1.5px + today = le jour actuel

Visualisable en un coup d'œil, économise du vertical vs cards empilées.

Section "Mes séances" dédiée. Liste des workout templates, chaque row avec name + meta (exos · durée · muscles). Tap → detail view. Compteur "4" en mono rouge dans le sec-label.

Card IA en dashed accent. Différenciée visuellement des workouts (dashed = action, pas contenu). Kanji 鍛 en head, description concrète, CTA "Générer ›" aligné accent.

"+ Nouveau" en top-right. Raccourci vers le builder. Compact, ne perturbe pas la hiérarchie du hub.

Jours en format 1=Monday. Fix du pain point #8 de l'audit Plan Builder (DB utilise 0=Sunday, UI hardcoded 1=Monday). On affiche "Lun → Mar → Mer → Jeu → Ven → Sam → Dim" consistent avec CLAUDE.md §7.

Builder · Rempli — Light

Stepper uniforme · tap-only partout

Zéro +/− sur toutes les cellules. Sets, Reps, Poids, Repos passent tous en tap-only. Les +/− prenaient de la place, fragmentaient l'interaction et créaient une incohérence visuelle entre les cellules. Solution unifiée : une cellule = un label + une valeur tappable.

Border-bottom dashed partout. Signale de façon uniforme que chaque valeur est un input éditable. L'affordance est cohérente quelle que soit la cellule.

Valeur 18px mono, bien visible. Augmentée de 14px à 18px maintenant qu'il n'y a plus les boutons +/− qui concurrençaient l'espace. La lecture des valeurs est plus rapide.

4 cellules égales (grid 1fr×4). Même largeur pour Sets / Reps / Poids / Repos. Pas de cellule privilégiée, pas de hiérarchie visuelle arbitraire.

State "editing" (vu sur Ex.01, 80 kg). Valeur en accent rouge, border-bottom solide 1.5px. Indique que le keyboard natif est ouvert pour cette cellule.

Keyboard natif iOS/Android. Chaque cellule ouvre le bon type de keyboard au tap : numérique pour Sets/Reps/Poids, picker MM:SS ou numérique pour Repos. Validation au blur ou au retour/OK.

Nouveautés du Plan Builder

Schedule summary avec day badges. Fix du pain point #10 (pas de edit-in-place). Chaque ligne a 2 actions : ⇄ Swap (rouvre le picker inline pour changer la séance) et ✕ Remove (retire le jour du plan). Plus besoin de remove + re-add.

Warning ambre "Activation". Fix du pain point #11 (activate-on-create side effect silencieux). Le user voit clairement que son plan actuel sera désactivé AVANT de tapoter le CTA. Seulement affiché si un plan actif existe déjà.

Jours en format Lun-Dim (1=Monday). Fix du pain point #8 (day value inconsistency). L'UI affiche Lun en position 1, Dim en position 7, consistent avec CLAUDE.md §7. Le mapping vers la DB (0=Sunday) se fait au save.

CTA "Activer ce plan" au lieu de "Save & activate plan". Plus direct, plus court, en français.

Compteur "Jours d'entraînement 4" en mono rouge dans le sec-label. Feedback immédiat.
