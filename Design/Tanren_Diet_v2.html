Changements vs v1

Description hero ajustée pour refléter l'approche "nutritionniste IA" — on met en avant les 20 questions et le fait que l'IA s'inspire des goûts perso, pas juste un calcul calorique.

Feature 02 "Repas basés sur tes goûts" remplace "Macros alignées muscu". Le prompt demande explicitement les 5 repas préférés — c'est LA différenciation vs MyFitnessPal.

Feature 03 "7 jours thématisés" nouveau. Le prompt demande un thème par jour (Mediterranean Monday, Tex-Mex Tuesday). C'est un élément fun qui vend l'expérience.

CTA meta "5 minutes · 4 étapes" au lieu de 3 minutes. Le prompt demande 20 inputs au lieu de 10, donc 5 min est plus honnête.

Nouveautés v2 alignées sur le prompt

Bloc "theme du jour" au-dessus des macros. Visuellement c'est un bloc border-left accent avec titre du thème ("Wok Wednesday 🥢"). Fait partie du caractère du plan — chaque jour a son identité culinaire.

Dessert optionnel avec border-left ambre (distinct des repas principaux en rouge). Le prompt demande explicitement "one optional dessert per day" — il fallait le matérialiser visuellement.

Cal cibles adaptées. 1680 kcal en semaine, 1880 kcal samedi/dimanche (refeed). Le prompt gère la vitesse de perte avec déficit de 500 kcal, donc affichage cohérent avec un homme de 82kg en cut steady.

Bloc "Liste de courses" en fin de page (NOUVEAU). Point d'entrée clair vers l'écran Groceries. Icône kanji 買 (achat), compteur "12/47 cochés", semaine en cours, progress bar accent, chevron rouge ›. Tap sur le bloc entier ouvre la liste. Border-left 3px accent comme les meal cards → signature visuelle Tanren cohérente.

Chevron "Plan ›" en top-right ouvre l'écran "Régénérer" (section 09) au lieu de l'édition manuelle — puisque le prompt ne permet pas d'éditer, seulement de régénérer.

Pourquoi ces inputs matchent le prompt

Type de job. Le prompt est explicite : "most online calculators underestimate TDEE significantly for manual workers". Demander le job type permet à l'IA de choisir le bon multiplicateur — Sedentary 1.2, Lightly 1.375, Moderately 1.55, Very 1.725, Extremely 1.9. Trois options suffisent (bureau / debout / manuel), l'IA combine avec la fréquence d'exercice.

Fréquence + type d'exercice. Segmented pour la fréquence (0-1 / 2-3 / 4+), champ libre pour le type. "Muscu full body + cardio" vs "CrossFit quotidien" donnera des multiplicateurs différents même à fréquence identique.

Sommeil & stress. Le prompt en a besoin pour recommander du magnésium (sleep issues) et adapter les règles perso. 7h + stress moyen → IA peut suggérer "dors 8h les soirs précédant tes grosses séances".

Alcool. Le prompt insiste : "factor those calories into the relevant days rather than ignoring them". 1-5 verres/sem = peut être intégré samedi sans casser le plan. 6+ = règle perso dédiée.

Pourquoi 4 questions sur les snacks

Snacks actuels. Champ libre. L'IA lit "chips, chocolat noir" et propose un swap qui scratch the same itch — "pop-corn maison 30g + carrés chocolat 85% au lieu des chips". Pas d'alternative triste.

Motivation (faim/ennui/habitude). Change radicalement le conseil. Si c'est la faim → augmenter les protéines au repas précédent. Si c'est l'ennui → règles comportementales (brosser les dents, thé). Si c'est l'habitude → remplacer le trigger.

Sucré/salé/les deux. Oriente les swaps. "Les deux" → alterner skyr-miel et olives-fromage.

Grignotage nocturne. Si "souvent" → règle perso dédiée + repas du soir plus protéiné pour la satiété. Si "jamais" → pas besoin de snack banque.

Modal spec

Slide up depuis le bas. Drag handle en haut (petit trait gris), swipe down pour fermer. Bouton ✕ en top-right pour accessibility. Pattern iOS standard.

Pas de preview vidéo inline. On garde le modal pur et compact : macros → ingrédients → recette → CTA YouTube. Pas de player embed qui charge du contenu tiers dans l'app.

Bouton "Voir la recette sur YouTube" en bas, style distinctif : border-left 3px rouge YouTube + icône play rouge carrée + label + nom de chaîne et durée en meta. Tap ouvre l'app YouTube native (ou Safari en fallback) sur la vidéo directement. Pas de fiction, c'est juste un deep link.

Pas de bouton "J'ai mangé". Le check-in ne sert pas le core use-case (suivi d'un plan généré par IA avec calories fixes). Simplifie l'interaction et évite d'encourager des comportements de tracking obsessionnel.

Pourquoi ce design

Compteur en haut, bloc dédié. Tu as choisi l'option "bloc dédié" — c'est le plus transparent. "1 / 2" en gros chiffres mono, barre avec 2 slots (1 rempli, 1 vide), date de reset explicite. Le user voit en 0.3s ce qu'il peut faire.

Bloc explicatif "Pourquoi limité à 2 ?". Important pour justifier la contrainte. Sans explication, le user râle. Avec "chaque génération IA coûte", il comprend et accepte. Transparence = confiance.

Deux actions possibles.

→ "Régénérer à l'identique" : utilise les mêmes inputs du dernier intake. Juste nouveau plan. Pour le user dont les préférences n'ont pas changé mais qui s'ennuie du plan actuel.

→ "Modifier mes réponses" : reprend le flow 4 steps de l'intake. Pour celui qui veut ajuster (nouveau poids, nouvelle fréquence d'exercice, nouvelles préférences).

Les deux consomment −1 crédit. Affiché explicitement à droite de chaque action. Pas de piège — le user sait avant de cliquer.

"Supprimer mon plan" en ghost rouge. Action destructive, discrète mais accessible. Ne consomme pas de crédit (on peut supprimer pour repartir de zéro gratuitement — la contrainte est sur la génération IA, pas la suppression locale).
