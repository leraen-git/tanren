Delete Account — Light

Pattern 2-step destruction

Étape 1 (pas mockupée) : modal depuis Profile avec message "Es-tu sûr ?" + bouton "Continuer". Étape 2 (celle-ci) : confirmation forte avec saisie obligatoire de SUPPRIMER. Cette friction volontaire est un standard UX pour les actions irréversibles (GitHub, Stripe, etc.). Le bouton reste rouge pour rappeler le danger, mais l'input à remplir empêche les suppressions accidentelles. Note spéciale : les séances déjà publiées sur Apple Santé ne sont pas supprimées côté Apple — c'est cohérent avec le modèle de HealthKit (on ne peut pas écrire un delete sur des données qu'on a écrites, l'utilisateur doit le faire manuellement dans Santé).
