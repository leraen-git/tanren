Ajouts clés sur l'écran existant

View toggle en haut. Segmented control Liste / Stats, le user switche entre les deux vues. Filtres et summary strip restent partagés entre les deux modes.

Section headers temporels. "Cette semaine / Semaine dernière / Il y a 2 semaines / Mars 2026 / Février 2026". Le regroupement temporel est automatique selon la date de la session. Au-delà de 4 semaines, on bascule sur des headers par mois. Un count de séances s'affiche à droite.

PR badge. Sur les session cards qui ont eu au moins un PR, un badge flottant top-right avec l'icône trophée + le nombre de PR. La card prend aussi une border-left rouge 3px pour être identifiable instantanément dans la liste.

Bouton search top-right. Recherche par nom de séance (ex. "Push Day") — utile quand l'historique devient long.

Composition de la vue Stats

Block 1 : Volume total. Grande valeur en Black weight (38px) + trend vs période précédente. Le trend est coloré : vert si progression, ambre si flat, rouge si régression.

Block 2 : Heatmap GitHub-style. Grille 7 lignes (Lun-Dim) × 12 colonnes (semaines, le plus récent à droite). 5 niveaux de rouge selon le volume : vide, 20%, 40%, 70%, 100%. Légende "Moins / Plus" en bas. Labels des mois au-dessus. Immédiatement compréhensible par les gens qui connaissent GitHub.

Block 3 : Volume hebdomadaire. Bar chart 12 semaines, la dernière en accent plein (semaine en cours). Permet de voir la tendance sur 3 mois en un coup d'œil.

Block 4 : Records récents. 3-5 derniers PR avec nom exercice, date relative ("il y a 2 jours") et valeur. Icône trophée rouge. Cliquable pour naviguer vers l'Exercise Detail.

Filtres période étendus. En mode Stats, on ajoute "1 an" dans les filtres — ça fait sens vu qu'on peut voir plus de données agrégées. En mode Liste on garde la période courte.

Différence entre les deux empty states

Empty state global. Gros kanji 錬 (96px opacity 85%), tagline "Historique vide", titre explicatif, description motivante avec la phrase signature "Rep après rep, ton histoire se construit", et CTA primary rouge pour démarrer une séance. Centré verticalement. Ne montre PAS les filtres ni le summary (pas de sens).

Empty state filtré. Plus petit et moins dramatique. Les filtres et le summary restent visibles (mais à 0 / 0). L'empty state est minimal : petite icône carrée en border, titre court "Pas de séance pour ce filtre", une desc courte et un lien ghost "Réinitialiser les filtres" en rouge. Le user comprend que c'est son filtre, pas un problème de data.

Structure de Session Detail

Hero avec corner rouge. Reprend le pattern TodayCard mais en mode "replay" : kanji 鍛 錬 stamp, nom séance gros, date complète avec heure, 3 stats (durée / volume / séries), muscle chips en bas. Tap sur le header "⋯" ouvre un menu (éditer, supprimer, export).

PR banner si applicable. Apparaît UNIQUEMENT s'il y a eu au moins un PR pendant cette séance. Fond rouge teinté + border-left 3px + kanji 鍛 + résumé des PR. Si pas de PR, on skip direct à la liste exercices.

Liste exercices avec grille séries. Chaque exercice a son bloc : header avec numéro (arabe) + nom + volume total de l'exo. En dessous, une grille 4 colonnes (# / Reps / Charge / Repos) avec les valeurs réelles. Les cells de la série où un PR a été battu sont en couleur accent avec un mini badge "PR".

Grille typographique JetBrains Mono. Les valeurs numériques dans la grille utilisent la font mono pour alignement parfait et feel "dashboard technique". Contraste avec le reste de l'app en Barlow Condensed.

Actions en bas. Deux boutons côte à côte : "Rejouer" (outline) crée une nouvelle session à partir du même template, "Partager" (primary rouge) ouvre la ShareCard 9:16 prérendu avec les stats de CETTE séance.
