Tanren · Health Integration v2
Apple Santé & Health Connect
Mockup unifié corrigé selon les 7 limitations techniques identifiées. iOS read permissions ne sont plus affichées comme toggles (Apple ne les expose pas). État "indisponible" géré explicitement. Révocation via redirection settings système.

✅ Corrections vs v1

  1. Read permissions iOS ne sont plus des toggles → bloc "Permissions à gérer dans Réglages" (Apple n'expose pas l'état de read auth fiablement)
  2. État "Apps santé indisponibles" ajouté pour Android sans Health Connect / iOS sans HealthKit
  3. Révocation via modal explicative + Linking.openSettings(), plus de fake toggle off qui ne fait rien
  4. Bloc d'info HealthKit pour expliquer les limitations Apple à l'user de façon transparente
  5. Compatible avec useTheme() existant (pas de tokens hardcodés)

Architecture

Une seule UX, deux providers selon plateforme : Platform.OS === 'ios' → Apple Santé via @kingstinct/react-native-healthkit · 'android' → Health Connect via react-native-health-connect.

Trigger : modal présentée après la 1ère séance complétée (flag MMKV tanren-health-prompted-v1).

Permissions iOS : seul write_workouts a un état détectable. Read permissions sont déléguées aux Réglages système.

Permissions Android : Health Connect expose toutes les perms via getGrantedPermissions(), donc UI complète avec toggles individuels.

Theme system — IMPORTANT

Tous les composants doivent consommer le hook useTheme() existant de l'app, pas les tokens inline du mockup HTML. Les valeurs CSS variables ci-dessus (--bg, --accent, etc.) sont des indicateurs visuels — l'implémentation React Native doit puiser dans le ThemeContext.

Avant toute UI, lire : grep -rn "useTheme\|ThemeContext\|ThemeProvider" apps/mobile/src et utiliser les tokens trouvés.

Transitions et états

Tap sur "Activer la synchronisation" :

iOS → modal Tanren se ferme → iOS native sheet apparaît (gérée par OS, pas Tanren) → user accepte/refuse → retour au screen précédent → état "Connecté" si write_workouts granted.
Android → modal Tanren se ferme → Health Connect ouvre son propre activity → user accepte/refuse → callback Tanren → état mis à jour.

Tap sur le toggle principal "off" sur l'écran connecté :

Affiche la modal de révocation (écran 6). User tape "Ouvrir Réglages" → Linking.openSettings(). À son retour dans Tanren, le service refait getPermissionState() et met à jour l'UI.

Tap sur "Sync maintenant" :

Spinner remplace le label le temps de la sync (1-3s typique). Met à jour la timestamp "Dernière sync".
